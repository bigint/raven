title: "Custom Domains"

Custom domains let you serve the Raven API from your own domain (e.g., `ai.yourcompany.com`) instead of `api.raven.dev`. This provides a branded API experience and simplifies migration if you ever switch providers.

## Adding a Custom Domain

<Steps>
  <Step title="Navigate to Settings">
    Go to **Settings > Custom Domains** in the dashboard.
  </Step>
  <Step title="Add Your Domain">
    Enter your custom domain (e.g., `ai.yourcompany.com`).
  </Step>
  <Step title="Configure DNS">
    Add the CNAME record shown in the dashboard to your DNS provider.
  </Step>
  <Step title="Verify">
    Raven verifies the DNS record automatically and provisions an SSL certificate.
  </Step>
</Steps>

<Note>
  Custom domains require the **Pro**, **Team**, or **Enterprise** plan.
</Note>

## DNS Verification via Cloudflare

Raven uses Cloudflare for domain verification and SSL provisioning. When you add a domain:

1. A unique verification token is generated
2. You add a CNAME record pointing to Raven's proxy
3. Raven verifies the DNS record through Cloudflare
4. An SSL certificate is automatically provisioned and renewed

### Required DNS Record

Add a CNAME record to your DNS provider:

| Type | Name | Value |
|------|------|-------|
| CNAME | `ai` (or your subdomain) | `proxy.raven.dev` |

<Warning>
  Custom domains must be unique across all Raven organizations. If a domain is already claimed by another organization, you will receive a conflict error.
</Warning>

## Domain Statuses

Your custom domain progresses through these states:

| Status | Description |
|--------|-------------|
| **Pending Verification** | DNS record has not been verified yet |
| **Verified** | DNS record confirmed, SSL being provisioned |
| **Active** | Domain is fully active and serving requests |
| **Failed** | Verification failed -- check your DNS configuration |

## Using Your Custom Domain

Once your domain is active, use it in place of `api.raven.dev`:

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3001/v1/chat/completions \
  -H "Authorization: Bearer rk_live_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```typescript TypeScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "rk_live_abc123...",
  baseURL: "http://localhost:3001/v1"
});

const response = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: "Hello!" }]
});
```

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="rk_live_abc123...",
    base_url="http://localhost:3001/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)
```

</CodeGroup>

## Organization Binding

Each custom domain is bound to a specific organization. When a request arrives on a custom domain:

1. Raven resolves the domain to the owning organization
2. The virtual key in the request is verified against that organization
3. If the key belongs to a different organization, the request is rejected with `403 Forbidden`

This ensures that custom domains cannot be used to access resources from other organizations.

## SSL/TLS

Raven automatically provisions and renews SSL certificates for custom domains via Cloudflare. You do not need to manage certificates manually.

- **Protocol** -- TLS 1.2+ enforced
- **Certificate** -- Auto-provisioned and auto-renewed
- **HSTS** -- Strict Transport Security headers included

## Deleting a Custom Domain

You can delete a custom domain at any time from the **Settings > Custom Domains** page. After deletion:

- The domain is immediately released and can be claimed by another organization
- Requests to the deleted domain will fail
- The Cloudflare hostname entry is cleaned up
- Existing API clients should be updated to use `api.raven.dev` or another active custom domain
