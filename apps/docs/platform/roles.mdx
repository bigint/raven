Raven uses role-based access control (RBAC) to manage permissions at the organization level. Every member is assigned one of five roles, each with a specific set of permissions.

## The Five Roles

| Role | Description | Use Case |
|------|-------------|----------|
| **Owner** | Full control over the organization | Founders, billing admins |
| **Admin** | Administrative access to all resources | Platform engineers, DevOps leads |
| **Manager** | Create and manage resources, limited admin | Team leads, project managers |
| **Developer** | Read access plus key creation | Application developers |
| **Viewer** | Read-only access to dashboards | Stakeholders, auditors |

## How Permission Checking Works

When a user makes a request, Raven's RBAC middleware:

1. Extracts the user's organization role from the session
2. Looks up the required permission for the endpoint
3. Checks if the role includes that permission
4. Returns `403 Forbidden` if the permission is missing

```
Request: POST /providers
Required: providers:write
User role: developer
Result: 403 Forbidden -- Insufficient permissions. Required: providers:write
```

## Inviting Members

<Steps>
  <Step title="Go to Team">
    Navigate to the **Team** section in the dashboard.
  </Step>
  <Step title="Click Invite">
    Enter the user's email address.
  </Step>
  <Step title="Select a Role">
    Choose from Owner, Admin, Manager, Developer, or Viewer.
  </Step>
  <Step title="Send Invitation">
    The user receives an email invitation to join the organization.
  </Step>
</Steps>

## Changing Roles

Organization owners and admins can change member roles from the **Team** page:

<Warning>
  There must always be at least one Owner in an organization. You cannot remove the Owner role from the last owner.
</Warning>

## Owner vs. Admin

The key differences between Owner and Admin:

| Capability | Owner | Admin |
|-----------|-------|-------|
| Manage billing | Yes | No |
| Delete organization | Yes | No |
| Export audit logs | Yes | No |
| All other permissions | Yes | Yes |

The Owner role has every permission in the system. The Admin role has nearly all permissions except billing management, organization deletion, and audit log export.
