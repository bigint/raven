Raven supports enterprise single sign-on (SSO) via SAML and OpenID Connect (OIDC) for centralized identity management.

## Supported Protocols

| Protocol | Description |
|----------|-------------|
| **OIDC** | OpenID Connect — Modern, token-based SSO |
| **SAML** | Security Assertion Markup Language — Enterprise standard |
| **OAuth** | GitHub and Google OAuth (built-in) |

## Configuring SSO

### OIDC Setup

<Steps>
  <Step title="Navigate to Settings">
    Go to **Settings > SSO** in the dashboard.
  </Step>
  <Step title="Select OIDC">
    Choose OpenID Connect as the provider.
  </Step>
  <Step title="Enter configuration">
    Provide your OIDC issuer URL, client ID, and client secret.
  </Step>
  <Step title="Test the connection">
    Verify SSO works by signing in with your identity provider.
  </Step>
</Steps>

### Built-in OAuth

GitHub and Google OAuth are available out of the box:

| Provider | Required Config |
|----------|----------------|
| **GitHub** | `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET` |
| **Google** | `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` |

## Just-in-Time (JIT) Provisioning

When JIT provisioning is enabled, users are automatically created in Raven when they sign in via SSO for the first time. No manual invitation required.

## Session Management

Raven tracks active sessions with:

- **IP address** — Where the session was created
- **User agent** — Browser/device information
- **Created at** — When the session started

Users can view and revoke their active sessions from their profile.

## Plan Availability

SSO/OIDC configuration is available on **Enterprise** plans. GitHub and Google OAuth are available on all plans.
