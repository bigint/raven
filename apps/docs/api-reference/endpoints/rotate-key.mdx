## Rotate Virtual Key

`POST /keys/:id/rotate`

Generates a new key secret while keeping the same configuration. The old key is immediately invalidated.

### Path Parameters

| Parameter | Description |
|-----------|-------------|
| `id` | Virtual key ID |

### Example Request

```bash
curl -X POST http://localhost:3001/keys/key_abc123/rotate \
  -H "Authorization: Bearer rk_live_..."
```

### Example Response

```json
{
  "id": "key_abc123",
  "name": "Production App",
  "key": "rk_live_newkey789...",
  "prefix": "rk_live_newk",
  "environment": "live",
  "rateLimitRpm": 100,
  "rateLimitRpd": 10000,
  "createdAt": "2025-01-15T10:00:00Z",
  "updatedAt": "2025-01-20T14:30:00Z"
}
```

<Warning>
  The old key stops working immediately. Update your application with the new key before clients attempt to use it.
</Warning>
