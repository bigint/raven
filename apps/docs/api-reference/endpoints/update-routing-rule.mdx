## Update Routing Rule

`PATCH /routing-rules/:id`

Updates an existing routing rule.

### Path Parameters

| Parameter | Description |
|-----------|-------------|
| `id` | Routing rule ID |

### Request Body

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | No | New name |
| `priority` | integer | No | New priority |
| `condition` | object | No | New condition |
| `targetModel` | string | No | New target model |
| `isEnabled` | boolean | No | Enable or disable |

### Example Request

```bash
curl -X PATCH http://localhost:4000/routing-rules/rule_abc123 \
  -H "Authorization: Bearer rk_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "isEnabled": false
  }'
```

### Example Response

```json
{
  "id": "rule_abc123",
  "name": "Migrate GPT-4 to GPT-4o",
  "priority": 1,
  "isEnabled": false,
  "updatedAt": "2025-01-20T14:30:00Z"
}
```
