---
title: "IP Allowlists"
description: "Restrict API access to trusted IP addresses and CIDR ranges."
---

IP allowlists let you restrict which IP addresses can make requests through your Raven virtual keys. When an allowlist is configured, requests from IP addresses outside the list are rejected with a `403 Forbidden` response.

## How It Works

When an organization has IP allowlist rules configured, every incoming proxy request is checked:

1. The client IP is extracted from `X-Forwarded-For`, `X-Real-IP`, or `CF-Connecting-IP` headers
2. The IP is compared against each CIDR rule in the organization's allowlist
3. If the IP matches any rule, the request proceeds
4. If no rules match, the request is rejected

<Note>
  If no IP allowlist rules are configured for your organization, all IP addresses are permitted. The allowlist only takes effect once you add at least one rule.
</Note>

## Adding IP Rules

Navigate to **Security > IP Allowlists** in the dashboard to manage your rules.

<Steps>
  <Step title="Go to IP Allowlists">
    Navigate to **Security > IP Allowlists** in the dashboard sidebar.
  </Step>
  <Step title="Add a Rule">
    Click **Add Rule** and enter a name and CIDR range.
  </Step>
  <Step title="Save">
    The rule takes effect within 60 seconds due to Redis caching.
  </Step>
</Steps>

### CIDR Notation

Rules use [CIDR notation](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing) to specify IP addresses and ranges.

| Example | Description |
|---------|-------------|
| `203.0.113.10/32` | Single IPv4 address |
| `203.0.113.0/24` | Class C subnet (256 addresses: `203.0.113.0` -- `203.0.113.255`) |
| `10.0.0.0/8` | Class A private network (16.7M addresses) |
| `192.168.1.0/24` | Common private subnet |
| `2001:db8::1/128` | Single IPv6 address |
| `2001:db8::/32` | IPv6 prefix |

### Single IP Address

To allow a single IP address, use a `/32` prefix for IPv4 or `/128` for IPv6:

```
203.0.113.42/32
```

If you omit the prefix length, Raven assumes `/32` for IPv4 and `/128` for IPv6 (exact match).

### Subnet Range

To allow an entire subnet, specify the appropriate prefix length:

```
# Allow all IPs in 10.20.30.0 - 10.20.30.255
10.20.30.0/24

# Allow all IPs in 172.16.0.0 - 172.16.255.255
172.16.0.0/16
```

### Class C Example

A `/24` block is the most common subnet size for office networks:

```
# Your office network (256 addresses)
198.51.100.0/24
```

This allows any IP from `198.51.100.0` through `198.51.100.255`.

## IPv4 and IPv6

Raven supports both IPv4 and IPv6 addresses. IPv6 addresses are fully expanded before comparison, so shorthand notation like `::1` works correctly.

```
# IPv4
203.0.113.0/24

# IPv6
2001:0db8:85a3::/48
```

<Warning>
  IPv4 and IPv6 rules are evaluated independently. An IPv4 rule will never match an IPv6 client and vice versa.
</Warning>

## Enabling and Disabling Rules

Individual rules can be toggled on and off without deleting them. This is useful for:

- **Temporary access** -- Enable a contractor's IP range during an engagement, then disable it
- **Debugging** -- Temporarily disable a rule to test connectivity
- **Staged rollouts** -- Add rules in a disabled state, then enable them when ready

## Redis Caching

IP allowlist rules are cached in Redis for **60 seconds** to minimize database queries on every request. This means:

- Changes to your allowlist take effect within 60 seconds
- High-throughput API traffic does not create database load from allowlist checks
- Each organization's rules are cached independently with the key pattern `ip-allowlist:{orgId}`

## Per-Organization Isolation

IP allowlists are scoped to individual organizations. Each organization manages its own set of rules, and there is no cross-organization visibility or inheritance.

## Common Configurations

### Office Network Only

Allow traffic only from your corporate network:

```
# Primary office
198.51.100.0/24

# VPN exit node
203.0.113.50/32
```

### Cloud Infrastructure

Allow traffic from your cloud provider's IP ranges:

```
# AWS us-east-1 NAT gateway
52.70.100.0/24

# GCP fixed external IP
35.200.50.10/32
```

### Development and Production

Combine developer IPs with production server ranges:

```
# Production servers
10.0.0.0/16

# Developer VPN
172.16.0.0/12

# CI/CD runner
203.0.113.200/32
```

## Error Responses

When a request is blocked by the IP allowlist, Raven returns:

```json
{
  "error": {
    "message": "Request denied: IP address not in allowlist",
    "code": "FORBIDDEN"
  }
}
```

The response status code is `403 Forbidden`. If Raven cannot determine the client IP at all, it returns:

```json
{
  "error": {
    "message": "Unable to determine client IP address",
    "code": "FORBIDDEN"
  }
}
```
