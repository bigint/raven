---
title: "IP Allowlists"
description: "Restrict API access to specific IP addresses or CIDR ranges."
---

IP allowlists let you restrict which IP addresses can make API requests to your organization.

## How It Works

When an IP allowlist is configured, only requests from listed IP addresses or CIDR ranges are accepted. All other requests receive a `403 Forbidden` response.

## Creating an Allowlist

Navigate to **Security > IP Allowlists** in the dashboard.

| Field | Description |
|-------|-------------|
| **Name** | Descriptive name for the entry |
| **CIDR** | IP address or CIDR range (e.g., `203.0.113.0/24`) |

## CIDR Format

| Format | Matches |
|--------|---------|
| `203.0.113.1/32` | Single IP address |
| `203.0.113.0/24` | 256 addresses (203.0.113.0 - 203.0.113.255) |
| `10.0.0.0/8` | Class A private network |

## Use Cases

- **Office access** — Only allow requests from your office IP range
- **VPN restriction** — Require all API access to go through your VPN
- **Server allowlisting** — Only allow requests from your production servers
- **Compliance** — Meet regulatory requirements for network access control
