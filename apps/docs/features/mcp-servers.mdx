Raven supports the [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) -- an open standard for connecting AI models to external tools and data sources. Register MCP servers in Raven to manage access control, monitor health, and track tool usage across your organization.

## What is MCP?

MCP is a protocol that standardizes how AI models interact with external systems. MCP servers expose **tools**, **resources**, and **prompts** that models can use during conversations. Instead of building custom integrations for each tool, MCP provides a unified interface.

Common MCP server examples:
- **Database access** -- Query PostgreSQL, MongoDB, or other databases
- **File systems** -- Read and write files in cloud storage
- **API integrations** -- Connect to Slack, GitHub, Jira, and other services
- **Custom tools** -- Build domain-specific tools for your organization

## Registering an MCP Server

<Steps>
  <Step title="Navigate to MCP Servers">
    Go to **MCP Servers** in the dashboard sidebar.
  </Step>
  <Step title="Add a Server">
    Enter the server name, URL, and transport type.
  </Step>
  <Step title="Configure Access Control">
    Specify which virtual keys and teams are allowed to use this server.
  </Step>
  <Step title="Set Health Check Interval">
    Configure how often Raven should check the server's health (default: 60 seconds).
  </Step>
</Steps>

### Via the API

```bash
curl -X POST https://api.raven.dev/v1/mcp-servers \
  -H "Authorization: Bearer rk_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "GitHub Tools",
    "url": "https://mcp.example.com/github",
    "transport": "streamable-http",
    "description": "GitHub repository management tools",
    "capabilities": ["tools", "resources"],
    "accessControl": {
      "allowedKeys": ["key_abc123"],
      "allowedTeams": ["team_engineering"]
    },
    "healthCheckInterval": 60
  }'
```

## Transport Types

MCP supports multiple transport mechanisms for server communication.

| Transport | Description | Use Case |
|-----------|-------------|----------|
| `stdio` | Standard input/output streams | Local tools, CLI-based servers |
| `sse` | Server-Sent Events over HTTP | Web-based servers with streaming |
| `streamable-http` | HTTP with streaming support | Production web services |

## Access Control

Control which virtual keys and teams can access each MCP server.

```json
{
  "accessControl": {
    "allowedKeys": ["key_abc123", "key_def456"],
    "allowedTeams": ["team_engineering", "team_support"]
  }
}
```

| Field | Description |
|-------|-------------|
| `allowedKeys` | Virtual keys that can invoke this server's tools (empty = all keys) |
| `allowedTeams` | Teams that can access this server (empty = all teams) |

<Info>
  When both `allowedKeys` and `allowedTeams` are empty, the MCP server is accessible to all keys and teams in the organization.
</Info>

## Health Monitoring

Raven periodically checks the health of each registered MCP server.

| Status | Description |
|--------|-------------|
| `active` | Server is responding and operational |
| `inactive` | Server is not currently active |
| `error` | Server is failing health checks |

Configure the health check interval per server (in seconds). The default is 60 seconds.

```json
{
  "healthCheckInterval": 60,
  "lastHealthCheck": "2026-03-15T10:30:00Z",
  "status": "active"
}
```

When a server transitions to `error` status, Raven emits a `provider.health.degraded` event that you can subscribe to via [Webhooks](/features/webhooks).

## Capabilities Tracking

Each MCP server declares its capabilities -- the types of features it supports.

| Capability | Description |
|-----------|-------------|
| `tools` | Server exposes callable tools |
| `resources` | Server provides data resources |
| `prompts` | Server offers prompt templates |

The `toolCount` field tracks how many tools are registered with each server.

## Server Configuration

| Field | Description | Default |
|-------|-------------|---------|
| `name` | Display name for the server | Required |
| `url` | Server endpoint URL | Required |
| `transport` | Communication protocol | `stdio` |
| `description` | What the server does | Empty |
| `capabilities` | Supported feature types | `[]` |
| `healthCheckInterval` | Seconds between health checks | 60 |
| `isEnabled` | Whether the server is active | `true` |
| `metadata` | Custom key-value metadata | `{}` |

## Catalog Integration

MCP servers can be published to the [AI Catalog](/features/catalog) for discovery by other team members. This is useful for sharing internally-built tool servers across the organization.
