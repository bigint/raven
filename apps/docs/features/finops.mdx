The FinOps module provides advanced financial operations capabilities for AI spend -- going beyond basic analytics with cost attribution by arbitrary tags, anomaly detection using statistical methods, and cost forecasting with budget overrun predictions.

## Cost Attribution

Track costs by arbitrary metadata tags to attribute spend to specific customers, features, teams, or any other dimension.

### How It Works

When making requests through the gateway, attach metadata tags to attribute costs:

```bash
curl -X POST https://api.raven.dev/v1/chat/completions \
  -H "Authorization: Bearer rk_live_abc123" \
  -H "X-Raven-Tags: customer=acme,feature=chatbot,team=engineering" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### Attribution Dimensions

| Tag | Use Case |
|-----|----------|
| `customer` | Per-customer cost tracking for billing or chargebacks |
| `feature` | Cost per product feature (chat, search, summarization) |
| `team` | Department-level cost allocation |
| `environment` | Production vs staging vs development spend |
| `model` | Automatic -- always tracked |
| `provider` | Automatic -- always tracked |

### Querying Cost Breakdowns

Get a cost breakdown by any tag over a date range:

```bash
curl https://api.raven.dev/v1/finops/costs/breakdown \
  -H "Authorization: Bearer rk_live_abc123" \
  -G \
  -d "tag=customer" \
  -d "start=2026-03-01" \
  -d "end=2026-03-15"
```

## Anomaly Detection

Raven uses a rolling Z-score algorithm (Welford's online method) to detect cost anomalies in real-time. After building a baseline of at least 20 requests, the system flags unusual spending patterns.

### Anomaly Types

| Type | Trigger | Severity |
|------|---------|----------|
| `cost_spike` | Single request cost is 3+ standard deviations above the mean | High (3-5 SD) or Critical (5+ SD) |
| `recursive_loop` | 5+ identical requests at the same cost in rapid succession | High |
| `budget_drain` | Rapid budget consumption detected | Medium |

### How Z-Score Detection Works

For each virtual key, Raven maintains a running mean and variance of request costs:

1. Each new request cost is compared against the historical baseline
2. A Z-score is calculated: `(currentCost - mean) / standardDeviation`
3. If the Z-score exceeds 3.0, a **high** severity anomaly is detected
4. If the Z-score exceeds 5.0, it escalates to **critical** severity

<Warning>
  Anomaly detection requires at least 20 historical data points before it becomes active. During the baseline-building phase, all requests are treated as normal.
</Warning>

### Recursive Loop Detection

The system also detects when the same cost appears repeatedly in rapid succession -- a common sign of an AI agent stuck in a loop:

- Monitors the last 10 request costs per key
- If 5 or more are identical (within $0.0001), triggers a `recursive_loop` anomaly

## Cost Forecasting

Project monthly costs based on daily spend trends and predict budget overruns before they happen.

```bash
curl https://api.raven.dev/v1/finops/costs/forecast \
  -H "Authorization: Bearer rk_live_abc123" \
  -G \
  -d "budget=1000"
```

Response:

```json
{
  "data": {
    "period": "2026-03",
    "currentSpend": 450.00,
    "projectedSpend": 930.00,
    "dailyRate": 30.00,
    "daysRemaining": 16,
    "budgetMax": 1000.00,
    "projectedOverrun": false,
    "confidence": 0.85
  }
}
```

### Overrun Prediction

When a budget is provided, the forecast calculates whether current spending trends will exceed the limit:

- **Projected overrun date** -- The estimated date when the budget will be exhausted
- **Confidence level** -- Increases with more data points (full confidence after 14 days)

## API Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /finops/costs/breakdown` | Cost breakdown by tag over a date range |
| `GET /finops/costs/forecast` | Monthly cost forecast with optional budget |
| `GET /finops/costs/daily` | Daily cost time series (default: last 30 days) |
| `GET /finops/costs/models` | Cost comparison across models |
| `GET /finops/costs/providers` | Cost comparison across providers |

<Note>
  Cost attribution data is retained for 90 days in Redis. For longer-term analysis, export data using the API endpoints.
</Note>
