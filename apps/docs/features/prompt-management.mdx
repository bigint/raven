Prompt management lets you create, version, and organize reusable prompt templates. Instead of hardcoding prompts in your application, store them in Raven and reference them by name.

## Creating a Prompt

Navigate to **Prompts** in the dashboard and click **Create Prompt**.

| Field | Description |
|-------|-------------|
| **Name** | A unique identifier for the prompt (e.g., `summarizer`, `translator`) |
| **Content** | The prompt template text |
| **Variables** | Placeholders that get replaced at runtime |

## Versioning

Every prompt supports multiple versions. Only one version can be **active** at a time — this is the version used when the prompt is referenced.

<Steps>
  <Step title="Create a new version">
    Edit the prompt content and save it as a new version.
  </Step>
  <Step title="Test the version">
    Use the chat interface to test the new version before activating it.
  </Step>
  <Step title="Activate the version">
    Set the new version as active. All requests referencing this prompt will use the new version.
  </Step>
  <Step title="Rollback if needed">
    Reactivate a previous version instantly if the new one doesn't perform well.
  </Step>
</Steps>

## Using Prompts in Requests

Reference a prompt template in your API request:

```json
{
  "model": "gpt-4o",
  "messages": [
    {
      "role": "system",
      "content": "{{prompt:summarizer}}"
    },
    {
      "role": "user",
      "content": "Summarize this article..."
    }
  ]
}
```

Raven resolves `{{prompt:summarizer}}` to the active version's content before forwarding the request to the provider.

## Template Variables

Use variables in your prompts for dynamic content:

```
You are a {{tone}} assistant that helps users with {{domain}}.
Always respond in {{language}}.
```

Pass variable values in the request metadata to substitute them at runtime.

## Benefits

- **Centralized management** — Update prompts without redeploying your app
- **Version history** — Track changes and roll back instantly
- **A/B testing** — Test prompt variations using experiments
- **Team collaboration** — Multiple team members can edit and review prompts
