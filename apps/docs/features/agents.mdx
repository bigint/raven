title: "Agent Identities"

Agent identities are first-class entities in Raven that represent AI agents operating within your organization. Each agent gets its own identity with scoped capabilities, budget limits, delegation permissions, and anomaly monitoring -- giving you governance over autonomous AI systems.

## Creating an Agent Identity

<Steps>
  <Step title="Navigate to Agents">
    Go to **Agents** in the dashboard sidebar.
  </Step>
  <Step title="Define the Agent">
    Set a name, description, and type for the agent.
  </Step>
  <Step title="Configure Capabilities">
    Specify which models and tools the agent is allowed to use.
  </Step>
  <Step title="Set Budget Limits">
    Assign a spending limit and period to control costs.
  </Step>
  <Step title="Link a Virtual Key">
    Optionally associate the agent with a specific virtual key.
  </Step>
</Steps>

### Via the API

```bash
curl -X POST https://api.raven.dev/v1/agents \
  -H "Authorization: Bearer rk_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "support-bot",
    "type": "semi_autonomous",
    "description": "Customer support agent with access to knowledge base",
    "capabilities": {
      "allowedModels": ["gpt-4o", "claude-4-sonnet"],
      "allowedTools": ["search_kb", "create_ticket"],
      "maxConcurrentRequests": 5,
      "maxTokensPerRequest": 4096,
      "maxCostPerRequest": 0.10
    },
    "budgetMax": 100.00,
    "budgetPeriod": "monthly"
  }'
```

## Agent Types

| Type | Description | Use Case |
|------|-------------|----------|
| `autonomous` | Operates independently, makes decisions without human input | Background data processing, scheduled reports |
| `semi_autonomous` | Operates within defined boundaries, escalates edge cases | Customer support, content moderation |
| `tool` | Invoked by other agents or systems, no independent decision-making | Database queries, API calls, calculations |

## Status Management

| Status | Description |
|--------|-------------|
| `active` | Agent can make requests through the gateway |
| `suspended` | Agent is temporarily disabled; requests are rejected |
| `revoked` | Agent is permanently disabled; cannot be reactivated |

## Capability Scoping

Each agent has a capabilities configuration that controls what it can do.

```json
{
  "capabilities": {
    "allowedModels": ["gpt-4o", "gpt-4o-mini"],
    "allowedTools": ["search", "calculator", "create_ticket"],
    "maxConcurrentRequests": 10,
    "maxTokensPerRequest": 100000,
    "maxCostPerRequest": 1.00
  }
}
```

| Capability | Description | Default |
|-----------|-------------|---------|
| `allowedModels` | Models the agent can use (empty = all) | `[]` (all models) |
| `allowedTools` | Tools the agent can invoke (empty = all) | `[]` (all tools) |
| `maxConcurrentRequests` | Maximum parallel requests | 10 |
| `maxTokensPerRequest` | Token limit per individual request | 100,000 |
| `maxCostPerRequest` | Cost cap per individual request | $1.00 |

## Per-Agent Budgets

Each agent can have its own budget, independent of the organization and team budgets.

| Field | Description |
|-------|-------------|
| `budgetMax` | Maximum spend for the budget period |
| `budgetPeriod` | Reset period (`daily` or `monthly`) |
| `budgetSpent` | Current spend in the active period |

When an agent's budget is exhausted, its requests are blocked until the next period -- even if the organization or team budget still has headroom.

<Warning>
  Agent budgets are part of the budget hierarchy. A request from an agent is checked against the organization budget, team budget, key budget, and agent budget. If any level is exceeded, the request is blocked.
</Warning>

## Delegation Chains

Agents can delegate tasks to other agents, creating hierarchical workflows.

```
Orchestrator (depth 0)
  |
  +-- Researcher (depth 1)
  |     |
  |     +-- Web Scraper (depth 2)
  |
  +-- Writer (depth 1)
  |
  +-- Reviewer (depth 1)
```

| Field | Description | Default |
|-------|-------------|---------|
| `canDelegate` | Whether the agent can invoke other agents | `false` |
| `delegationDepth` | Current depth in the delegation chain | 0 |
| `maxDelegationDepth` | Maximum allowed delegation depth | 3 |
| `parentAgentId` | The agent that delegated to this one | `null` |

<Info>
  The `maxDelegationDepth` limit prevents runaway delegation chains. If an agent at depth 3 tries to delegate, the request is rejected.
</Info>

## Anomaly Detection

Raven monitors agent behavior for anomalies and emits events when unusual patterns are detected.

| Anomaly | Description | Event |
|---------|-------------|-------|
| Cost spikes | Single request cost far exceeds the agent's historical average | `agent.anomaly.detected` |
| Token explosions | Unusually large token consumption | `agent.anomaly.detected` |
| Budget exceeded | Agent exhausted its budget | `agent.budget.exceeded` |
| Recursive loops | Repeated identical requests (potential infinite loop) | `agent.anomaly.detected` |

Subscribe to agent events via [Webhooks](/features/webhooks) to get notified when anomalies occur.

## Activity Tracking

Each agent tracks its last activity timestamp (`lastActiveAt`), enabling you to identify stale or abandoned agents.

```bash
curl https://api.raven.dev/v1/agents \
  -H "Authorization: Bearer rk_live_abc123"
```

The response includes status, budget utilization, and last active time for each agent identity.
