---
title: "Policies"
description: "Define and enforce governance policies with deterministic rules and compliance mapping."
---

Policies are governance-as-code rules that enforce organizational standards on every AI request. Unlike guardrails (which focus on content filtering), policies operate at the infrastructure level -- controlling which models can be used, enforcing cost limits, restricting regions, and mapping rules to compliance frameworks.

## Policy-as-Code

Each policy contains one or more rules that are evaluated against every request. Policies support versioning, status management, and compliance framework mapping.

```json
{
  "name": "Production Safety Policy",
  "scope": "organization",
  "status": "active",
  "version": 1,
  "complianceFrameworks": ["soc2", "hipaa"]
}
```

A policy progresses through three statuses:

| Status | Description |
|--------|-------------|
| `draft` | Policy is being developed and is not enforced |
| `active` | Policy is live and enforced on all matching requests |
| `archived` | Policy is retired and no longer evaluated |

## Policy Scopes

Policies are scoped to control their blast radius. Narrower scopes override broader ones.

| Scope | Applies To | Example |
|-------|-----------|---------|
| `platform` | All organizations | Platform-wide model restrictions |
| `organization` | Entire org | Block deprecated models org-wide |
| `team` | Specific team | Engineering team cost limits |
| `key` | Single virtual key | Limit a staging key to GPT-4o-mini only |

## Rule Types

Each policy rule has a type that determines how it is evaluated.

| Type | Description | Use Case |
|------|-------------|----------|
| `deterministic` | Exact pattern matching with guaranteed results | Regex, keyword, model allow/deny lists |
| `statistical` | Threshold-based evaluation | Cost anomaly detection, token limits |
| `ml_model` | ML-powered evaluation | Advanced content classification |

## Enforcement Levels

When a rule matches, the enforcement level determines the response.

| Level | Behavior |
|-------|----------|
| `block` | Reject the request immediately |
| `warn` | Allow the request, attach a warning |
| `log` | Silently record the match |
| `redact` | Remove or mask the matched content |
| `alert` | Allow the request, trigger an alert event |

## Condition Types

Policy rules support nine condition types that cover content, model, cost, and composite logic.

<AccordionGroup>
  <Accordion title="regex" icon="code">
    Match request content against a regular expression pattern.
    ```json
    { "type": "regex", "params": { "pattern": "\\bconfidential\\b" } }
    ```
  </Accordion>
  <Accordion title="pii" icon="user-shield">
    Detect PII in request content. Supports the same PII types as guardrails (SSN, email, phone, credit card, IP address).
    ```json
    { "type": "pii", "params": { "piiTypes": ["ssn", "creditCard"] } }
    ```
  </Accordion>
  <Accordion title="keyword" icon="font">
    Match against a list of keywords (case-insensitive).
    ```json
    { "type": "keyword", "params": { "keywords": ["internal", "secret"] } }
    ```
  </Accordion>
  <Accordion title="model_allowlist" icon="check">
    Only allow requests to specific models. Triggers when the requested model is **not** in the list.
    ```json
    { "type": "model_allowlist", "params": { "models": ["gpt-4o", "claude-4-sonnet"] } }
    ```
  </Accordion>
  <Accordion title="model_denylist" icon="ban">
    Block requests to specific models. Triggers when the requested model **is** in the list.
    ```json
    { "type": "model_denylist", "params": { "models": ["gpt-3.5-turbo"] } }
    ```
  </Accordion>
  <Accordion title="provider_region" icon="globe">
    Restrict requests to providers in specific geographic regions.
    ```json
    { "type": "provider_region", "params": { "regions": ["us", "eu"] } }
    ```
  </Accordion>
  <Accordion title="cost_threshold" icon="dollar-sign">
    Block requests estimated to exceed a cost threshold.
    ```json
    { "type": "cost_threshold", "params": { "maxCost": 0.50 } }
    ```
  </Accordion>
  <Accordion title="token_threshold" icon="text-width">
    Block requests that set `max_tokens` or `max_completion_tokens` above a limit.
    ```json
    { "type": "token_threshold", "params": { "maxTokens": 4096 } }
    ```
  </Accordion>
  <Accordion title="composite" icon="layer-group">
    Combine multiple conditions with `AND` or `OR` logic.
    ```json
    {
      "type": "composite",
      "operator": "AND",
      "params": {
        "conditions": [
          { "type": "keyword", "params": { "keywords": ["production"] } },
          { "type": "cost_threshold", "params": { "maxCost": 1.00 } }
        ]
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Compliance Framework Mapping

Each policy rule can be mapped to specific compliance framework controls. This enables automatic evidence generation for audits.

```json
{
  "name": "Block unencrypted PHI transmission",
  "type": "deterministic",
  "enforcement": "block",
  "condition": { "type": "pii", "params": { "piiTypes": ["ssn"] } },
  "complianceMap": {
    "hipaa": "164.312.e.1",
    "soc2": "CC6.1",
    "gdpr": "Art.32"
  }
}
```

Supported frameworks: **SOC 2 Type II**, **HIPAA**, **GDPR**, and **EU AI Act**. See [Compliance](/features/compliance) for details.

## Policy Inheritance

Policies follow a hierarchy from broad to narrow scope:

1. **Platform** policies apply to all organizations.
2. **Organization** policies apply to the entire org.
3. **Team** policies apply to a specific team's keys.
4. **Key** policies apply to a single virtual key.

All applicable policies are evaluated for every request. If **any** policy blocks the request, it is rejected regardless of other policies.

<Warning>
  Policy evaluation results are stored in the `policy_evaluations` table with the request ID, matched rule, enforcement action, and compliance evidence. This data powers compliance reporting.
</Warning>

## Creating a Policy via API

```bash
curl -X POST https://api.raven.dev/v1/policies \
  -H "Authorization: Bearer rk_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Restrict to approved models",
    "scope": "organization",
    "status": "active",
    "complianceFrameworks": ["soc2"],
    "rules": [
      {
        "name": "Approved models only",
        "type": "deterministic",
        "enforcement": "block",
        "condition": {
          "type": "model_allowlist",
          "params": {
            "models": ["gpt-4o", "claude-4-sonnet", "gemini-2.5-pro"]
          }
        },
        "complianceMap": { "soc2": "CC6.3" }
      }
    ]
  }'
```
