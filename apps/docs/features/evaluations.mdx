Evaluations let you systematically measure the quality of LLM outputs using built-in evaluators. Run evaluation batches against prompt/response pairs to score relevance, coherence, PII leakage, and toxicity -- then track quality trends over time.

## Built-in Evaluators

Raven includes four evaluators that score outputs on a 0.0 to 1.0 scale with pass/fail results.

| Evaluator | What It Measures | Pass Threshold |
|-----------|-----------------|----------------|
| `relevance` | How well the response relates to the prompt | 0.3 |
| `coherence` | Structural quality (sentence count, length, readability) | 0.5 |
| `pii_leakage` | Whether the response contains PII (email, SSN, credit card, phone, IP) | No PII detected |
| `toxicity` | Whether the response contains harmful language | No toxic patterns |

### Relevance

Measures keyword overlap between the prompt and response. Higher overlap indicates the response addresses the prompt's topic.

- Extracts significant words (4+ characters) from prompt and response
- Calculates overlap ratio
- Returns a score, feedback text, and keyword metrics

### Coherence

Evaluates structural quality of the response:

- Checks for proper sentence structure
- Validates average sentence length (3-50 words is considered reasonable)
- Rewards multi-sentence responses

### PII Leakage

Scans the response for personally identifiable information:

- Email addresses
- Social Security Numbers (SSN)
- Credit card numbers
- Phone numbers
- IP addresses

A response with any PII detection fails this evaluator. The score decreases by 0.3 for each type of PII detected.

### Toxicity

Scans for harmful language patterns including hate speech, violence, discrimination, and slurs. Each toxic pattern match reduces the score by 0.5.

## Creating an Evaluation Run

<Steps>
  <Step title="Navigate to Evaluations">
    Go to **Evaluations** in the dashboard.
  </Step>
  <Step title="Create a New Evaluation">
    Set a name, select the evaluator type, and choose the model to evaluate.
  </Step>
  <Step title="Add Test Cases">
    Provide prompt/response pairs to evaluate. You can also provide expected outputs for comparison.
  </Step>
  <Step title="Run the Evaluation">
    Start the evaluation run. Raven processes each test case and generates scores.
  </Step>
</Steps>

### Via the API

```bash
curl -X POST https://api.raven.dev/v1/evaluations \
  -H "Authorization: Bearer rk_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Support bot quality check",
    "evaluatorType": "relevance",
    "model": "gpt-4o",
    "config": {}
  }'
```

## Evaluation Run Statuses

| Status | Description |
|--------|-------------|
| `pending` | Evaluation is queued but not yet started |
| `running` | Evaluation is actively processing test cases |
| `completed` | All test cases have been evaluated |
| `failed` | Evaluation encountered an error |

## Viewing Results

Each evaluation run produces:

| Field | Description |
|-------|-------------|
| **Overall score** | Average score across all test cases (0.0 - 1.0) |
| **Pass count** | Number of test cases that passed |
| **Fail count** | Number of test cases that failed |
| **Sample count** | Total test cases evaluated |

### Per-Result Details

Each individual result includes:

```json
{
  "input": "What is the capital of France?",
  "actualOutput": "The capital of France is Paris.",
  "expectedOutput": "Paris",
  "score": 0.95,
  "passed": "true",
  "feedback": "Response appears relevant to the prompt",
  "metrics": {
    "keywordOverlap": 4,
    "promptKeywords": 5,
    "responseKeywords": 6
  },
  "tokenCount": 42,
  "latencyMs": 230,
  "cost": 0.0004
}
```

## Running All Evaluators

You can run all four evaluators against a prompt/response pair at once to get a comprehensive quality assessment:

```json
{
  "relevance": { "score": 0.85, "passed": true },
  "coherence": { "score": 0.90, "passed": true },
  "piiLeakage": { "score": 1.00, "passed": true },
  "toxicity": { "score": 1.00, "passed": true }
}
```

## Custom Evaluators

Beyond the built-in evaluators, you can create custom evaluation criteria by specifying configuration in the evaluation's `config` field. Custom evaluators can define their own scoring logic and pass/fail thresholds.

## Integration with Experiments

Evaluations can be paired with [Experiments](/features/experiments) to add quality metrics alongside cost and latency data. This enables you to find the cheapest model that still meets your quality bar.

<Note>
  Evaluation results are stored with full provenance -- including the input, output, token count, latency, and cost for each test case. Use this data to track quality regressions over time.
</Note>
