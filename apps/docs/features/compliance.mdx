title: "Compliance"
description: "Automate compliance evidence generation for SOC 2, HIPAA, GDPR, and EU AI Act."
---

Raven automates compliance evidence generation by mapping your gateway configurations -- guardrails, policies, and audit logs -- to specific controls in industry-standard compliance frameworks. Instead of manually gathering evidence for audits, Raven continuously evaluates your configuration and generates compliance reports on demand.

## Supported Frameworks

| Framework | Version | Controls | Focus Area |
|-----------|---------|----------|------------|
| **SOC 2 Type II** | 2022 | 8 controls | Security, monitoring, access control |
| **HIPAA** | 2023 | 8 controls | PHI protection, encryption, audit controls |
| **GDPR** | 2018 | 9 controls | Data protection, privacy by design, cross-border transfers |
| **EU AI Act** | 2024 | 7 controls | Risk management, transparency, human oversight |

## How It Works

<Steps>
  <Step title="Configure Guardrails and Policies">
    Set up guardrails (PII detection, content filtering) and policies (model restrictions, cost limits) in your organization.
  </Step>
  <Step title="Map Rules to Controls">
    When creating policy rules, map them to specific compliance controls using the `complianceMap` field.
  </Step>
  <Step title="Raven Evaluates Continuously">
    Every request that passes through the gateway is evaluated against your policies. Results are stored as compliance evidence.
  </Step>
  <Step title="Generate Reports">
    Request a compliance report for any framework. Raven checks your configurations and generates evidence for each control.
  </Step>
</Steps>

## Evidence Generation

Raven generates evidence automatically by checking your organization's configuration state. Each control receives a status:

| Status | Meaning |
|--------|---------|
| **Met** | Control requirements are fully satisfied by active configurations |
| **Partial** | Some configurations exist but are not specifically mapped to the control |
| **Not Met** | No configurations address this control |

## Compliance Reports

Request a compliance report via the API:

```bash
curl https://api.raven.dev/v1/compliance/frameworks/soc2/report \
  -H "Authorization: Bearer rk_live_abc123"
```

The report includes an overall score and per-control evidence:

```json
{
  "framework": "soc2",
  "frameworkName": "SOC 2 Type II",
  "overallScore": 75,
  "controls": [
    {
      "control": "CC6.1",
      "controlName": "Logical Access",
      "status": "met",
      "evidence": "Access control policies are configured and mapped to this control.",
      "source": "policy_engine",
      "collectedAt": "2026-03-15T10:00:00Z"
    }
  ],
  "generatedAt": "2026-03-15T10:00:00Z"
}
```

The **overall score** is calculated as: `(met + partial * 0.5) / total * 100`.

## Available APIs

| Endpoint | Description |
|----------|-------------|
| `GET /compliance/frameworks` | List all supported frameworks |
| `GET /compliance/frameworks/:id` | Get framework details and controls |
| `GET /compliance/frameworks/:id/report` | Generate a compliance report |

<Note>
  Compliance reports reflect your current configuration state. Adding guardrails and mapping policy rules to controls will improve your compliance score over time.
</Note>
