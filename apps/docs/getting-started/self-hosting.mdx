---
title: "Self-Hosting"
description: "Deploy Raven on your own infrastructure with Docker Compose."
---

Raven can be self-hosted using Docker Compose. The platform consists of four services: the API server, the web dashboard, PostgreSQL, and Redis.

## Prerequisites

- Docker and Docker Compose installed
- A domain name (optional, for production)
- SMTP credentials for email (optional)

## Quick Start

### 1. Clone the Repository

```bash
git clone https://github.com/ravenplatform/raven.git
cd raven
```

### 2. Configure Environment

Create a `.env` file from the example:

```bash
cp .env.example .env
```

Edit the `.env` file with your configuration:

```env
# Required
DATABASE_URL=postgresql://raven:raven@postgres:5432/raven
REDIS_URL=redis://redis:6379
BETTER_AUTH_SECRET=your-secret-at-least-16-chars
BETTER_AUTH_URL=http://localhost:3001
APP_URL=http://localhost:3000
NEXT_PUBLIC_API_URL=http://localhost:3001

# Encryption (min 32 characters)
ENCRYPTION_SECRET=your-32-character-encryption-key-here

# Optional: OAuth
GITHUB_CLIENT_ID=
GITHUB_CLIENT_SECRET=
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=

# Optional: Email
RESEND_API_KEY=

# Optional: Billing
LEMONSQUEEZY_API_KEY=
LEMONSQUEEZY_WEBHOOK_SECRET=

# Optional: Observability
OTEL_ENABLED=false
OTEL_EXPORTER_OTLP_ENDPOINT=
OTEL_EXPORTER_OTLP_HEADERS=
```

### 3. Start Services

```bash
docker compose up -d
```

This starts:

| Service | Port | Description |
|---------|------|-------------|
| **web** | 3000 | Next.js dashboard |
| **api** | 3001 | Hono API server |
| **postgres** | 5432 | PostgreSQL 17 database |
| **redis** | 6379 | Redis 7 cache and rate limiting |

### 4. Run Migrations

```bash
docker compose exec api pnpm db:migrate
```

### 5. Access the Dashboard

Open [http://localhost:3000](http://localhost:3000) and create your first account.

## Production Deployment

For production deployments, ensure you:

<Warning>
  Never use default secrets in production. Generate strong random values for `BETTER_AUTH_SECRET` and `ENCRYPTION_SECRET`.
</Warning>

### Security Checklist

- Set strong values for `BETTER_AUTH_SECRET` (min 16 chars) and `ENCRYPTION_SECRET` (min 32 chars)
- Configure CORS by setting `APP_URL` to your actual domain
- Enable HTTPS via a reverse proxy (nginx, Caddy, etc.)
- Set `NODE_ENV=production`
- Configure proper database credentials
- Set up Redis with authentication if exposed

### Resource Recommendations

| Service | CPU | Memory | Storage |
|---------|-----|--------|---------|
| API | 2 cores | 2 GB | — |
| Web | 1 core | 1 GB | — |
| PostgreSQL | 2 cores | 4 GB | 50 GB SSD |
| Redis | 1 core | 256 MB | — |

### Kubernetes

A Helm chart is available for Kubernetes deployments. See the `helm/` directory in the repository for configuration options.

```bash
helm install raven ./helm/raven \
  --set api.env.DATABASE_URL=postgresql://... \
  --set api.env.REDIS_URL=redis://... \
  --set api.env.BETTER_AUTH_SECRET=...
```
