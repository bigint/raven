## Raven vs Portkey

[Portkey](https://portkey.ai) is a managed AI gateway with a strong feature set. The primary difference is deployment model.

**Choose Raven if:**

- You need to self-host on your own infrastructure
- You want full control over your data and prompts
- You prefer open-source with no vendor lock-in
- You want a single Docker image deployment

**Choose Portkey if:**

- You prefer a managed cloud service
- You need enterprise support and SLAs

## Raven vs LiteLLM

[LiteLLM](https://github.com/BerriAI/litellm) is a popular open-source Python proxy for LLM providers.

**Choose Raven if:**

- You want a complete platform with a dashboard, analytics, and team management
- You need built-in guardrails (PII detection, prompt injection, content filtering)
- You want audit logs and webhooks out of the box
- You prefer a single-binary Docker deployment over managing Python dependencies

**Choose LiteLLM if:**

- You want maximum provider coverage (100+ providers)
- Your stack is Python-centric and you want a Python library
- You need only the proxy layer without a UI

## Raven vs Helicone

[Helicone](https://helicone.ai) is an observability platform for LLM applications.

**Choose Raven if:**

- You need an actual proxy (routing, caching, guardrails), not just logging
- You want budget controls and rate limiting on API keys
- You want to self-host for free

**Choose Helicone if:**

- Your primary need is observability and analytics
- You want a managed service with minimal setup
- You need advanced prompt evaluation tools

## Raven vs Keywords AI

[Keywords AI](https://keywordsai.co) is a managed LLM monitoring and proxy service.

**Choose Raven if:**

- You need to self-host and control your data
- You want open-source software you can audit and modify
- You need guardrails and content filtering

**Choose Keywords AI if:**

- You prefer a managed service with no infrastructure to maintain
- You need built-in prompt evaluation and A/B testing

## Why Self-Hosting Matters

With managed AI gateways, every prompt and response passes through a third-party server. For teams handling sensitive data, regulated industries, or internal-only applications, this creates unnecessary risk.

Raven runs entirely on your infrastructure. Your prompts, API keys, and response data never leave your servers. You get the same gateway capabilities without the data exposure.
