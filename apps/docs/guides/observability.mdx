Raven includes built-in observability through Prometheus-compatible metrics, OpenTelemetry distributed tracing, and a real-time event streaming system. These work together to give you full visibility into your AI gateway.

## Prometheus Metrics

Raven exposes a Prometheus-compatible metrics endpoint that can be scraped by any Prometheus-compatible monitoring system.

### Endpoint

```
GET /metrics
```

Returns metrics in the Prometheus text exposition format (`text/plain; version=0.0.4`).

### Available Metrics

#### Request Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `raven_requests_total` | Counter | `model`, `provider`, `status` | Total requests processed |
| `raven_request_duration_seconds` | Histogram | `model`, `provider` | Request latency distribution |

The histogram uses standard buckets: `0.005`, `0.01`, `0.025`, `0.05`, `0.1`, `0.25`, `0.5`, `1`, `2.5`, `5`, `10`, `30`, `60` seconds.

#### Token Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `raven_tokens_total` | Counter | `model`, `provider`, `direction` | Total tokens processed (`input` or `output`) |

#### Cost Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `raven_cost_dollars_total` | Counter | `model`, `provider` | Total cost in US dollars |

#### Cache Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `raven_cache_hits_total` | Counter | `model`, `type` | Cache hits (`exact` or `semantic`) |

#### Guardrail Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `raven_guardrail_triggers_total` | Counter | `rule`, `action` | Guardrail triggers by rule and action |

#### Fallback Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `raven_fallback_total` | Counter | `from`, `to`, `success` | Fallback events between providers |

#### Rate Limit Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `raven_rate_limit_exceeded_total` | Counter | `key_id` | Rate limit exceeded events |

### Example Prometheus Scrape Config

```yaml
scrape_configs:
  - job_name: "raven"
    scrape_interval: 15s
    static_configs:
      - targets: ["raven-api:3000"]
    metrics_path: "/metrics"
```

## OpenTelemetry Traces

Raven exports distributed traces using the **OTLP (OpenTelemetry Protocol)** JSON format. Traces provide request-level visibility into every step of the proxy pipeline.

### Configuration

Enable OpenTelemetry with environment variables:

```env
OTEL_ENABLED=true
OTEL_EXPORTER_OTLP_ENDPOINT=https://your-collector.example.com
OTEL_EXPORTER_OTLP_HEADERS=Authorization=Bearer your-token
OTEL_SERVICE_NAME=raven-gateway
OTEL_FLUSH_INTERVAL_MS=10000
```

### Span Attributes

Every proxy request creates a span with these attributes:

| Attribute | Type | Description |
|-----------|------|-------------|
| `raven.model` | string | Model used for the request |
| `raven.provider` | string | Provider that served the request |
| `raven.organization_id` | string | Organization ID |
| `raven.virtual_key_id` | string | Virtual key ID |
| `raven.input_tokens` | number | Input token count |
| `raven.output_tokens` | number | Output token count |
| `raven.cost` | number | Request cost in dollars |
| `raven.cache_hit` | boolean | Whether the response was cached |
| `raven.cache_type` | string | Cache type (`exact` or `semantic`) |
| `raven.guardrail_triggered` | boolean | Whether a guardrail was triggered |
| `raven.fallback_used` | boolean | Whether a fallback provider was used |
| `raven.latency_ms` | number | Total request latency |
| `raven.status_code` | number | HTTP status code |
| `raven.routing_strategy` | string | Routing strategy used |
| `raven.task_type` | string | Task type classification |

### Compatible Backends

Send traces to any OpenTelemetry-compatible backend:

| Backend | OTLP Endpoint Format |
|---------|---------------------|
| **Grafana Tempo** | `http://tempo:4318/v1/traces` |
| **Jaeger** | `http://jaeger:4318/v1/traces` |
| **Datadog** | `https://trace.agent.datadoghq.com/v1/traces` |
| **Honeycomb** | `https://api.honeycomb.io/v1/traces` |
| **New Relic** | `https://otlp.nr-data.net/v1/traces` |

## Event Streaming

Raven's event system provides real-time notifications for platform events via Redis Pub/Sub.

### Event Types

Events are organized by category:

#### Request Events

| Event | Description |
|-------|-------------|
| `request.completed` | A proxy request completed successfully |
| `request.failed` | A proxy request failed |
| `request.cached` | A response was served from cache |
| `request.fallback` | A fallback provider was used |

#### Guardrail Events

| Event | Description |
|-------|-------------|
| `guardrail.triggered` | A guardrail rule matched |
| `guardrail.blocked` | A request was blocked by a guardrail |
| `guardrail.warned` | A guardrail warning was attached |

#### Budget Events

| Event | Description |
|-------|-------------|
| `budget.threshold.reached` | A budget alert threshold was hit |
| `budget.exceeded` | A budget was fully exhausted |
| `budget.reset` | A budget counter was reset |

#### Provider Events

| Event | Description |
|-------|-------------|
| `provider.health.degraded` | A provider's health score dropped |
| `provider.health.recovered` | A provider's health recovered |
| `provider.outage.detected` | A provider outage was detected |

### Event Structure

Every event includes:

```json
{
  "id": "evt_abc123",
  "type": "request.completed",
  "organizationId": "org_xyz",
  "timestamp": "2025-01-15T10:30:00Z",
  "data": {
    "model": "gpt-4o",
    "latencyMs": 450,
    "cost": 0.003
  },
  "metadata": {
    "provider": "openai",
    "virtualKeyId": "key_abc",
    "requestId": "req_123"
  }
}
```

## Health Check

Raven exposes health check endpoints:

```bash
# Simple health check
GET /health

# Detailed health check with memory and uptime
GET /health/detailed
```

The detailed endpoint returns:

```json
{
  "status": "ok",
  "timestamp": "2025-01-15T10:30:00Z",
  "uptime": 86400,
  "memory": {
    "rss": 104857600,
    "heapTotal": 67108864,
    "heapUsed": 52428800
  }
}
```

## Integrating with Grafana

For self-hosted deployments, build Grafana dashboards using the Prometheus metrics:

- **Request throughput** -- `rate(raven_requests_total[5m])`
- **P95 latency** -- `histogram_quantile(0.95, rate(raven_request_duration_seconds_bucket[5m]))`
- **Token usage** -- `rate(raven_tokens_total[1h])` by direction
- **Cost rate** -- `rate(raven_cost_dollars_total[1h])`
- **Cache hit rate** -- `rate(raven_cache_hits_total[5m]) / rate(raven_requests_total[5m])`
- **Error rate** -- `rate(raven_requests_total{status=~"5.."}[5m])`
- **Guardrail trigger rate** -- `rate(raven_guardrail_triggers_total[5m])` by rule

## Integrating with Datadog

Send OTLP traces and Prometheus metrics to Datadog:

```env
OTEL_ENABLED=true
OTEL_EXPORTER_OTLP_ENDPOINT=https://trace.agent.datadoghq.com
OTEL_EXPORTER_OTLP_HEADERS=DD-API-KEY=your-datadog-api-key
```

Configure the Datadog agent to scrape the `/metrics` endpoint for Prometheus metrics.
